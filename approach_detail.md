## Overview

This document provides a complete, technically rigorous description of an end-to-end pipeline
for detecting hallucinated scientific summaries in a severely low-resource setting. The task
is binary: given two text summaries of a space-science article — one generated by a large
language model (potentially hallucinated) and one grounded in the source material — identify
which summary is fabricated.

The defining challenge of this competition is extreme data scarcity. Only 95 labelled
training pairs (190 individual summaries) are available for supervised learning, while the
test set contains 1,068 unlabelled pairs. This asymmetry between training and test size,
combined with the prohibition on pretrained neural components, rules out the dominant paradigm
of fine-tuning large models. Any classifier with more than approximately 100 effective
parameters will overfit the 95-sample training set and fail to generalise.

The pipeline responds to this constraint through four interlocking strategies:

- A compact, 33-dimensional feature representation grounded in linguistics and information
  theory, with each feature chosen for discriminative power and interpretability.
- Aggressively regularised ensemble classifiers that deliberately sacrifice capacity to
  gain generalisation.
- Swap consistency enforcement at inference time to eliminate positional bias without
  introducing additional parameters.
- Conservative one-round pseudo-labelling to incorporate high-confidence unlabelled
  test samples into retraining.

The resulting system achieves a private leaderboard score of 0.852, placing it among the
top-performing submissions under the no-pretrained-backbone constraint.

## Problem Formalisation

Let the input to the system be a collection of ordered text pairs. For each pair, define:

    x_A  =  summary text drawn from file_1 (first alphabetically)
    x_B  =  summary text drawn from file_2 (second alphabetically)
    y    in {1, 2}  =  ground truth label

where y = 1 denotes that x_A is the hallucinated (fake) summary and y = 2 denotes that x_B
is the hallucinated summary. The clean text is always the complement.

The supervised training set is:

    D_train = { (x_A^(i), x_B^(i), y^(i)) : i = 1, ..., 95 }

The unlabelled test set is:

    D_test = { (x_A^(j), x_B^(j)) : j = 1, ..., 1068 }

The goal is to learn a function:

    f : (x_A, x_B)  ->  {1, 2}

that minimises zero-one loss on D_test. Because y is never observed for test pairs during
training, the model must generalise from 95 labelled examples to 1,068 held-out predictions.

This ratio — approximately 11 test samples per training sample — makes overfitting the
dominant risk. Every architectural decision in the pipeline is evaluated against this
constraint. Specifically, the pipeline is designed to satisfy four necessary conditions
for generalisation in this regime:

Condition 1 — Feature compactness. The feature vector must have substantially fewer
dimensions than training samples. The target is approximately 33 dimensions (0.35 features
per sample), well below the critical threshold of 1.0.

Condition 2 — Strong regularisation. All learnable components must carry explicit
regularisation that prevents them from memorising training noise. This applies to both
the base classifiers and any auxiliary models such as the language model.

Condition 3 — No trainable embeddings. High-dimensional trainable representations
(word vectors, contextual embeddings) require orders of magnitude more training data than
is available here. They are excluded entirely.

Condition 4 — Conservative use of unlabelled data. Test data may only be incorporated
into training when confidence exceeds a threshold calibrated to minimise expected label noise.

## Phase 1 — Corpus-Level Preprocessing

This phase is entirely unsupervised. It runs before any labels are examined and processes
all available texts — both training and test — to construct three shared components:
a TF-IDF vectoriser, a Kneser-Ney language model, and a pooled corpus for vocabulary
calibration. Using the full corpus (train + test) for unsupervised steps is both valid
and necessary: it ensures that IDF weights and language model probabilities are computed
over the true marginal distribution of language in this task, rather than an unrepresentative
training-only subsample.

### 1.1 Pooled Corpus Construction

Procedure:

All summaries from both the training and test sets are collected and concatenated into a
single list, corpus_all, containing:

     190 texts  (95 pairs x 2 summaries)  from D_train
    2136 texts  (1068 pairs x 2 summaries) from D_test
    ---------------------------------------------------
    2326 texts  total

Justification:

TF-IDF inverse document frequency weights are computed as log(N / df_t), where N is the
total number of documents and df_t is the number of documents containing term t. If computed
on training texts alone (N = 190), terms that appear exclusively in the test set will have
undefined IDF, distorting the cosine similarity computation in Tower 3. Pooling resolves
this without using any label information.

Similarly, the language model will encounter out-of-vocabulary tokens at test time if
trained on training texts only. Pooling reduces this vocabulary gap, improving perplexity
estimates on test texts.

### 1.2 TF-IDF Vectoriser Training

A standard TfidfVectorizer is fitted on corpus_all with the following configuration:

| Parameter       | Value    | Rationale |
|-----------------|----------|-----------|
| analyzer        | 'word'   | Word-level tokenisation; character-level would produce an intractably large vocabulary. |
| ngram_range     | (1, 2)   | Unigrams and bigrams; captures both single-term frequency and local phrase structure. |
| max_features    | 5000     | Retains the 5000 most informative terms; reduces dimensionality while preserving coverage. |
| min_df          | 2        | Ignores terms appearing in fewer than 2 documents; removes hapax legomena and artefacts. |
| sublinear_tf    | True     | Applies 1 + log(tf) to raw term frequencies; dampens influence of very frequent terms. |

Important: The resulting 5000-dimensional TF-IDF vectors are not used directly as classifier
inputs. The high dimensionality would cause immediate overfitting. Instead, they are used
solely to compute a single scalar — the cosine similarity between the two texts in a pair
(Phase 2, Tower 3). This collapses 10,000 dimensions (two 5000-d vectors) into one number
without any learned compression.

The fitted vectoriser is stored and applied identically to all subsequent texts to ensure
consistency between training-time and test-time representations.

### 1.3 Language Model Training — Kneser-Ney Smoothed Trigram

Motivation:

A language model trained exclusively on clean (non-hallucinated) summaries learns the
statistical regularities of grounded scientific prose. When applied to a candidate text,
it assigns lower perplexity to fluent, specific, factually dense language and higher
perplexity to the hedged, repetitive, or vague language that characterises hallucinated
outputs. Perplexity thus becomes a proxy for "how much does this text resemble clean
scientific writing?"

Training data selection:

From the 95 labelled training pairs, exactly one clean text is extracted per pair:

- If y = 1 (A is fake): extract x_B as the clean text.
- If y = 2 (B is fake): extract x_A as the clean text.

This yields exactly 95 clean texts. Hallucinated texts are deliberately excluded. Including
them would cause the language model to assign lower perplexity to hallucination patterns,
undermining the discriminative signal.

N-gram order selection:

Trigrams (n = 3) are chosen as the model order. Higher-order models (n = 4 or 5) would
produce sparser count matrices on 95 short texts, causing most trigram contexts to have
zero observed continuations and forcing heavy reliance on lower-order backoff — effectively
degrading to a bigram model anyway. Bigrams (n = 2) lose important syntactic context.
Trigrams represent the optimal trade-off for this corpus size.

Kneser-Ney Smoothing:

Kneser-Ney (KN) smoothing is the state-of-the-art algorithm for n-gram language models.
It addresses the zero-probability problem for unseen n-grams by discounting observed counts
and redistributing the probability mass to unseen events, weighted by continuation
probability — the number of unique contexts in which a word appears.

The core recursion for an order-3 KN model is:

    P_KN(w3 | w1, w2)  =  max(C(w1,w2,w3) - delta, 0) / C(w1,w2)
                        +  lambda(w1,w2) * P_KN(w3 | w2)

where:

- C(w1,w2,w3) = raw trigram count
- delta in (0, 1) = fixed discount, typically 0.75
- lambda(w1,w2) = normalisation factor ensuring probabilities sum to 1, computed as
  (delta / C(w1,w2)) * |{w : C(w1,w2,w) > 0}|
- The recursion continues to bigrams and finally unigrams, where the base distribution
  is the continuation probability: P_KN(w) proportional to |{v : C(v,w) > 0}|

Implementation steps (class KneserNeyLM):

1. Tokenise each clean text into a word list, lowercased, with <BOS> and <EOS> boundary markers.
2. Accumulate counts: C_3[w1,w2,w3], C_2[w1,w2], C_1[w], and continuation counts.
3. Implement the recursive probability function with backoff as described above.
4. For a given text at inference time, compute the log probability of each token given
   its trigram history, sum, and normalise to perplexity:

    perplexity(text) = exp( -(1/N) * sum_i log P_KN(w_i | w_{i-2}, w_{i-1}) )

   where N is the number of tokens in the text.

Feature extraction from perplexity:

For each pair (x_A, x_B), four scalar features are derived:

| Feature      | Formula              | Directional Interpretation |
|--------------|----------------------|----------------------------|
| lm_ppl_a     | perplexity(x_A)      | Absolute surprise of Text A. |
| lm_ppl_b     | perplexity(x_B)      | Absolute surprise of Text B. |
| lm_ppl_diff  | ppl_a - ppl_b        | Positive -> A more surprising -> A more likely hallucinated. |
| lm_ppl_ratio | log(ppl_a/ppl_b+eps) | Scale-invariant, symmetric; eps prevents log(0). |

These four values form Tower 4 of the feature vector.

## Phase 2 — Feature Engineering

The feature vector for each text pair is constructed from four complementary towers, totalling
33 dimensions. Each tower captures a distinct aspect of the relationship between the two
texts: individual linguistic properties, cross-pair overlap, global lexical similarity, and
language model surprise. The feature extraction logic is implemented in a vectorised manner
using pandas and numpy for computational efficiency across large batches.

### Tower 1 — Per-Text Linguistic Features (24 features)

Extraction procedure:

Twelve scalar linguistic features are extracted independently for Text A and for Text B,
yielding two 12-dimensional vectors f_A and f_B. These are then transformed:

    delta  =  f_A - f_B          (12 features: signed difference)
    mean   =  (f_A + f_B) / 2   (12 features: pair-level average)

Concatenating delta and mean yields 24 features. This transformation is strictly superior
to using the raw 24-dimensional vector [f_A, f_B] because:

- For tree-based models: A single split on delta_k >= 0 is equivalent to asking "is feature
  k larger in A than in B?" — a single decision node that would require two nodes in the
  raw representation.
- For linear models: Deltas and means are orthogonal projections of the raw pair, producing
  a better-conditioned feature matrix.
- Parameter efficiency: The number of features is identical (24), but the representation
  is more aligned with what classifiers need to learn: comparative relationships.

Definitions of the 12 per-text features:

---

n_words — Total word count after splitting on whitespace.

Hallucination signal: Hallucinated summaries may be systematically shorter (truncated
generation) or longer (rambling, repetitive continuation). A consistent directional
difference within a pair is evidence of fabrication.

---

n_chars — Total character count including whitespace and punctuation.

Hallucination signal: Complements word count; also captures punctuation density and
average token length at a coarser granularity than avg_word_len.

---

n_sents — Sentence count, obtained by splitting on `.`, `!`, `?` and filtering empty strings.

Hallucination signal: Hallucinated texts may have unusual sentence boundary patterns —
either extremely long run-on constructions or unusually fragmented short sentences.

---

avg_word_len — Computed as n_chars / n_words.

Hallucination signal: Scientific prose contains long, technical vocabulary such as
"photodissociation" or "magnetohydrodynamic". Hallucinating models may substitute simpler,
shorter words, reducing mean word length across the text.

---

avg_sent_len — Mean number of words per sentence: n_words / n_sents.

Hallucination signal: Incoherent or rambling hallucinations often produce abnormally
long sentences by chaining clauses without strong logical connectives or clear transitions.

---

longest_sent — Maximum sentence length in words across all sentences in the text.

Hallucination signal: A single extremely long sentence is a strong signal for generation
failure or hallucinated rambling, even if the remainder of the text appears normal.

---

max_repeat — Count of the single most frequently occurring word in the text.
For example, if "the" appears 15 times, this value is 15.

Hallucination signal: Degenerate LLM outputs frequently exhibit pathological repetition
of specific tokens or short phrases, a well-documented failure mode of autoregressive
language models in low-temperature or poorly calibrated sampling regimes.

---

flesch — Flesch Reading Ease score, computed as:

    flesch = 206.835 - 1.015 * (words / sentences) - 84.6 * (syllables / words)

Syllables are estimated by counting maximal vowel groups (a, e, i, o, u, y) per word.
Lower scores indicate more complex, harder-to-read text typical of dense scientific prose;
higher scores indicate simpler language.

Hallucination signal: Hallucinating models may produce easier-to-read text, either because
they oversimplify technical content or because the generated prose lacks the structural
complexity of expert scientific writing.

---

hedge_ratio — Proportion of hedge words relative to total words.

The hedge lexicon includes: approximately, roughly, about, estimated, around, nearly,
possibly, perhaps, probably, likely, seemingly, apparently, supposedly, reportedly,
allegedly, might, may, could, suggest, suggests, suggested, indicate, indicates, indicated,
appear, appears, appeared, seem, seems, seemed.

Hallucination signal: A language model that lacks factual grounding expresses epistemic
uncertainty through frequent hedging. Clean scientific summaries use hedges sparingly
and purposefully; hallucinated summaries hedge excessively to cover fabricated claims.

---

vague_ratio — Proportion of vague quantifiers relative to total words.

The vague lexicon includes: some, many, several, various, numerous, few, diverse, multiple,
countless, certain, particular, significant, substantial, considerable, extensive, vast,
enormous, tremendous, huge, massive, incredible, amazing.

Hallucination signal: Grounded scientific text employs precise numerical values such as
"5.2 sigma significance" or "a redshift of z = 3.14". Hallucinated text substitutes vague
quantifiers for specific measurements it cannot fabricate accurately.

---

informal_ratio — Proportion of informal or colloquial words relative to total words.

The informal lexicon includes: wow, amazing, incredible, awesome, cool, great, fantastic,
wonderful, brilliant, super, nice, hope, hopefully, heres, lets, dont, wont, cant, isnt,
arent, wasnt, werent, im, ive, id, ill, youre, theyre, weve, theyve, think, thinking,
basically, literally, actually, honestly, totally, absolutely, definitely, obviously, clearly.

Hallucination signal: The register of scientific summary writing is formal and impersonal.
Hallucinating models, particularly those fine-tuned on conversational data, occasionally
allow colloquial phrasing to infiltrate their outputs.

---

specificity — Proportion of specific tokens to total tokens, where specific tokens are:

- Proper nouns: Words beginning with uppercase, not the first word of a sentence, and
  not in a standard stopword list.
- Acronyms: All-uppercase strings of length >= 2 consisting only of letters (e.g., NASA,
  ESA, UV).
- Decimal numbers: Strings containing a decimal point (e.g., 3.14, 0.001, 5.2).

Hallucination signal: Factually grounded scientific text is dense with named entities,
instrument names, mission designations, and precise numerical values. Hallucinated text
tends to be more generic, substituting vague descriptors for specific named items.

---

n_citations — Count of citation patterns, detected via:

- Bracket notation: regex \[\d+\] (e.g., [1], [23])
- DOI patterns: regex 10\.\d{4,9}/[-._;()/:A-Za-z0-9]+

Hallucination signal: Grounded scientific summaries frequently reference prior work. A
hallucinating model, having no access to the source document's reference list, rarely
produces correctly formatted citations.

---

### Tower 2 — Cross-Pair Features (4 features)

These four features compare the two texts directly, treating them as sets rather than
measuring their individual properties. They are symmetric by construction: f(A,B) = f(B,A).
All values lie in [0, 1].

jaccard_1gram — Unigram-level Jaccard similarity:

    J1 = |set(tokens_A) intersect set(tokens_B)| / |set(tokens_A) union set(tokens_B)|

Measures the proportion of shared vocabulary between the two summaries. A value near 1
indicates the texts discuss similar content using similar words; a value near 0 suggests
one text introduces vocabulary absent from the other — a hallucination indicator when
the divergence involves named entities or technical terms.

---

jaccard_2gram — Bigram-level Jaccard similarity, computed analogously over the set of
all consecutive word pairs in each text:

    J2 = |bigrams(A) intersect bigrams(B)| / |bigrams(A) union bigrams(B)|

Bigrams capture phrase-level structure. A low jaccard_2gram combined with a moderate
jaccard_1gram indicates that the texts share vocabulary but arrange it in different
collocational contexts — a subtler signal of content divergence.

---

entity_overlap — Named entity Jaccard similarity. Capitalised tokens are extracted from
each text, excluding the first word of each sentence (capitalised by convention),
single-character tokens, and members of a standard stopword list. The Jaccard similarity
of the resulting entity sets is computed:

    entity_overlap = |entities(A) intersect entities(B)| / |entities(A) union entities(B)|

Hallucination signal: A low entity overlap indicates that one text introduces named
entities — people, organisations, missions, instruments, celestial objects — not present
in the other. Since both texts purport to summarise the same article, a hallucinated text
may fabricate or omit key entities.

---

num_overlap — Numeric Jaccard similarity. Numeric strings (including decimals) are
extracted via regex \d+(?:\.\d+)?. Jaccard similarity over these numeric sets is computed:

    num_overlap = |numerics(A) intersect numerics(B)| / |numerics(A) union numerics(B)|

Hallucination signal: Precise numerical values — measurements, distances, temperatures,
redshifts, significance levels — are among the hardest facts to hallucinate correctly.
A low numeric overlap strongly suggests that one text has fabricated or omitted quantitative
claims from the source document.

---

### Tower 3 — TF-IDF Cosine Similarity (1 feature)

Using the pre-fitted TfidfVectorizer from Phase 1, both texts are transformed into sparse
5000-dimensional vectors v_A and v_B. Their cosine similarity is computed as:

    cos_sim(A, B) = (v_A . v_B) / (||v_A|| * ||v_B||)

Since TF-IDF vectors are non-negative, the result lies in [0, 1].

This single scalar provides a global measure of lexical and topical overlap that complements
the discrete Jaccard metrics in Tower 2. Cosine similarity is sensitive to term weighting
(rare terms matter more than common ones, via the IDF component), whereas Jaccard treats
all terms equally. The two measures are therefore complementary, not redundant.

Why collapse to one dimension? The full TF-IDF vectors have 5000 dimensions each. Feeding
them as features would produce a 10,000-dimensional input to classifiers trained on 95
samples — an extreme overfitting scenario. Compressing to a single cosine similarity
retains the most important global signal at zero additional parameter cost.

### Tower 4 — Language Model Perplexity Features (4 features)

The four perplexity features derived in Phase 1.3 are used directly as classifier inputs:

| Feature      | Description                              | Directional Interpretation |
|--------------|------------------------------------------|----------------------------|
| lm_ppl_a     | Perplexity of Text A under clean-text LM | Higher -> A deviates more from clean prose. |
| lm_ppl_b     | Perplexity of Text B under clean-text LM | Higher -> B deviates more from clean prose. |
| lm_ppl_diff  | ppl_a - ppl_b                            | Positive values suggest A is hallucinated. |
| lm_ppl_ratio | log(ppl_a / ppl_b + eps)                 | Scale-invariant; eps = 1e-8 prevents errors. |

Both lm_ppl_diff and lm_ppl_ratio encode the same directional comparison at different
scales. Including both provides classifiers with complementary views: linear models benefit
from the additive diff form, while tree-based models may prefer the log-ratio form.

## Phase 3 — Ensemble Training

### 3.1 Label Encoding

The competition ground truth label real_text_id is re-encoded as a binary target:

    y = 1  if A is fake  (real_text_id = 2, meaning B is real)
    y = 0  if B is fake  (real_text_id = 1, meaning A is real)

All classifiers are trained to output P(A is fake). A prediction of p >= 0.5 maps to
y = 1 (A fake); p < 0.5 maps to y = 0 (B fake). The final competition label is recovered
as real_text_id = 2 - y.

### 3.2 Model Selection and Regularisation

Three base models are used. The number three is deliberate: more models would require
additional hyperparameter configurations that cannot be reliably tuned on 95 samples.
The three chosen models have complementary inductive biases — tree-based non-linearity,
kernel-based margin maximisation, and linear probabilistic regression — ensuring that
their error patterns are partially uncorrelated, which is the prerequisite for ensemble
improvement over any individual model.

Model 1: LightGBM (Gradient-Boosted Decision Trees)

LightGBM is configured to produce an ensemble of near-stumps:

| Hyperparameter     | Value | Regularisation Effect |
|--------------------|-------|-----------------------|
| n_estimators       | 50    | Fewer trees reduces total model capacity. Default is 100. |
| max_depth          | 2     | Trees have at most 2 levels; each makes at most 3 binary decisions. |
| num_leaves         | 4     | Maximum 4 leaves (= 2^max_depth); each tree is very coarse. |
| learning_rate      | 0.05  | Small step size; slow learning prioritises generalisation. |
| subsample          | 0.7   | Uses 70% of rows per tree; reduces co-adaptation between trees. |
| colsample_bytree   | 0.6   | Uses 60% of features per tree; prevents single-feature dominance. |
| reg_alpha          | 5.0   | L1 penalty; pushes some leaf weights to exactly zero. |
| reg_lambda         | 10.0  | L2 penalty; shrinks all leaf weights toward zero. |
| min_child_samples  | 15    | Each leaf must contain >= 15 samples (~16% of training set). |

Model 2: Support Vector Machine with RBF Kernel

The SVM is configured with a very wide margin:

| Hyperparameter  | Value     | Regularisation Effect |
|-----------------|-----------|-----------------------|
| C               | 0.05      | Low C = large margin = high bias, low variance. Default is 1.0. |
| kernel          | 'rbf'     | Radial basis function; non-linear boundaries constrained by C. |
| class_weight    | 'balanced'| Adjusts for any imbalance in training class frequencies. |
| StandardScaler  | Applied   | SVM is scale-sensitive; features standardised to zero mean, unit variance. |

Model 3: Logistic Regression with L2 Penalty

| Hyperparameter  | Value     | Regularisation Effect |
|-----------------|-----------|-----------------------|
| C               | 0.05      | Strong L2 penalty; shrinks all coefficients toward zero. |
| penalty         | 'l2'      | Ridge regularisation; produces dense solutions. |
| solver          | 'lbfgs'   | Efficient for small datasets with dense feature matrices. |
| class_weight    | 'balanced'| Same as SVM. |
| StandardScaler  | Applied   | Logistic regression with L2 is also scale-sensitive. |

### 3.3 Repeated Stratified K-Fold Cross-Validation

Why not a held-out validation set?

With 95 training samples, a standard 80/20 train-validation split yields only 76 training
samples and 19 validation samples. A validation set of 19 samples produces extremely noisy
accuracy estimates (plus or minus 8-10 percentage points at 95% confidence). Cross-validation
distributes this estimation cost across all samples, providing a much more reliable
performance estimate at the cost of additional computation.

Configuration:

    n_splits   = 5     Each fold has 19 samples; model trained on 76.
    n_repeats  = 3     Three independent 5-fold splits with different random seeds.
    Total splits = 15  train/validation configurations.

Stratified splitting ensures that each fold preserves the proportion of A-fake and B-fake
labels present in the full training set, preventing any fold from being unrepresentative.

Procedure for each of the 15 splits:

1. Partition the 95 samples into train_idx (4 folds, 76 samples) and val_idx (1 fold, 19 samples).
2. For each base model:
   - Train on X[train_idx], y[train_idx].
   - Predict P(A fake) on X[val_idx].
   - Store predictions in oof_probs[val_idx, model_index].
3. After all 15 splits, each sample has appeared in the validation fold exactly 3 times
   (once per repeat). Average the 3 predictions per sample to obtain a stable out-of-fold
   (OOF) probability estimate for each model.

Ensemble OOF prediction:

For each training sample, the final ensemble OOF probability is:

    p_oof = (p_oof_lgb + p_oof_svm + p_oof_lr) / 3

Binary prediction: y_hat = 1 if p_oof >= 0.5, else y_hat = 0.

OOF accuracy is the proportion of training samples correctly classified. This is a nearly
unbiased estimate of test performance, up to distribution shift. Per-model OOF accuracies
are computed separately for diagnostic purposes:

| Model                     | Typical OOF Accuracy |
|---------------------------|----------------------|
| LightGBM alone            | ~86%                 |
| SVM alone                 | ~83%                 |
| Logistic Regression alone | ~84%                 |
| Ensemble (3-model avg)    | ~87-89%              |

The consistent 1-2% gain from ensembling over the best individual model confirms that the
three models make partially independent errors on this dataset, validating the ensemble
design choice.

### 3.4 Final Model Training and Ensemble Inference

After cross-validation, each base model is retrained on the full 95 training samples with
identical hyperparameters. No data is withheld. These final models are used for all test
set predictions.

For a new test pair, each model outputs a scalar probability P(A is fake). The ensemble
prediction is the arithmetic mean:

    p_ensemble = (p_LightGBM + p_SVM + p_LogisticRegression) / 3

Why arithmetic mean and not a weighted average or stacked meta-learner?

A weighted average requires learning optimal weights w1, w2, w3. Even with only 3 free
parameters, a second-level model trained on the 95 OOF predictions would have approximately
1 parameter per 32 samples — insufficient data to estimate weights reliably without
overfitting. The arithmetic mean has zero trainable parameters and has been shown empirically
to outperform learned combinations in low-data ensemble settings.

## Phase 4 — Swap Consistency

### Motivation

The training data assigns Text A as file_1 and Text B as file_2 based on alphabetical
ordering. If this ordering is correlated with the label (e.g., if file_1 tends to be the
hallucinated summary more often due to data collection artefacts), a classifier may learn
a spurious positional bias — predicting A as fake without attending to content.

More broadly, even without explicit positional bias in the training data, a classifier
trained on ordered pairs may develop asymmetric decision boundaries: it may respond
differently to f(A, B) versus f(B, A) in ways that do not reflect genuine content
differences. Swap consistency is the principled remedy for this failure mode.

### Algorithm

For each test pair (text_A, text_B):

Step 1. Compute the original prediction:

    p1 = P(A is fake | text_A, text_B)

using the full ensemble with the standard feature extraction.

Step 2. Construct the swapped pair (text_B, text_A) and compute:

    p_swapped = P(B is fake | text_B, text_A)

Note that in the swapped pair, text_B occupies the "A position" and text_A occupies the
"B position". Therefore the probability that the original text_A is fake, as estimated
from the swapped pair, is 1 - p_swapped.

Step 3. Average the two estimates:

    p_final = (p1 + (1 - p_swapped)) / 2

Properties of this procedure:

- Probability conservation: P_final(A fake) + P_final(B fake) = 1 exactly, as required
  for a valid binary probability.
- Bias elimination: Any positional bias in the classifier is cancelled — excess probability
  assigned to the "A position" text unconditionally is averaged out with the swapped
  prediction.
- Test-time augmentation: Averaging two partially independent predictions reduces variance,
  yielding a modest but consistent accuracy improvement of approximately 1-2%.
- Zero additional parameters: No new learnable components are introduced.

Implementation note: The function predict_proba_a_fake(text_A, text_B, use_swap=True)
recursively calls itself with use_swap=False on the swapped pair. This doubles inference
time but has no effect on the final model structure or training procedure.

## Phase 5 — Conservative Pseudo-Labelling

### Motivation and Risk Assessment

The 1,068 unlabelled test pairs represent a potentially substantial source of additional
training signal. Semi-supervised learning methods can exploit this data, but only under
carefully controlled conditions. The primary risk is label propagation error: if the model
assigns an incorrect pseudo-label with high confidence, that erroneous label enters the
training set and biases the retrained model, which then becomes more confident in its
incorrect predictions. This feedback loop can substantially degrade performance relative
to the supervised baseline.

The safeguard is a confidence threshold calibrated to make the expected label error rate
negligible. At a threshold of 0.98, the model must assign a probability >= 0.98 to one
class before a pseudo-label is accepted. If the model's calibration is approximately
correct, this implies an expected error rate of at most 2% among pseudo-labelled samples.

### Procedure

Step 1 — Baseline training.
Train the full ensemble on the 95 labelled pairs using the procedure described in Phase 3,
including swap consistency (Phase 4).

Step 2 — Test set inference.
Apply the swap-consistent ensemble to all 1,068 test pairs, obtaining:

    p_j = P(A is fake | x_A^(j), x_B^(j))  for each test pair j

Step 3 — Pseudo-label selection.
Apply the confidence thresholds:

    If p_j >= 0.98  ->  assign pseudo-label y_j = 1  (A is fake)
    If p_j <= 0.02  ->  assign pseudo-label y_j = 0  (B is fake)
    Otherwise       ->  discard (do not pseudo-label)

In typical runs, this selects approximately 50-100 test pairs (5-10% of the test set).

Step 4 — Dataset augmentation.
Combine the original 95 labelled pairs with the pseudo-labelled pairs:

    D_augmented = D_train union { (x_A^(j), x_B^(j), y_j) : j pseudo-labelled }

Step 5 — Retraining from scratch.
Retrain the complete pipeline — feature extraction, cross-validation, and ensemble —
on D_augmented using identical hyperparameters. Warm-starting is deliberately avoided:
starting from the weights of the baseline model would bias the retrained model toward its
own previous predictions, amplifying any systematic errors.

Step 6 — Final test set prediction.
Apply the retrained ensemble with swap consistency to all 1,068 test pairs to produce
the final submission labels.

### Design Decisions

Why only one round?

Multi-round pseudo-labelling with a decreasing threshold (e.g., 0.98 -> 0.95 -> 0.90)
adds progressively noisier labels in each successive round. Even if the first round is safe,
the second round uses a model trained on first-round pseudo-labels, which may have systematic
biases not reflected in its confidence scores. One round at a high threshold is the safest
strategy for a 95-sample supervised baseline.

Why retrain from scratch?

Retraining from scratch on D_augmented treats pseudo-labelled and true-labelled pairs
identically in terms of influence on the final model. An alternative — assigning lower
sample weight to pseudo-labelled pairs — would require choosing a weight value, introducing
a new hyperparameter that cannot be reliably tuned on 95 samples.

Performance impact:

Pseudo-labelling improved the OOF accuracy of the retrained ensemble by approximately 2
percentage points relative to the baseline trained on 95 samples only. This improvement
persisted on the private leaderboard, confirming genuine generalisation gain rather than
overfitting to the pseudo-label selection criterion.

## Design Principles and Anti-Overfitting Measures

Every component of this pipeline was designed under the hard constraint of 95 labelled
training samples. The table below provides a comprehensive summary of the anti-overfitting
measures employed, their implementations, and the reasoning behind each choice.

| Principle | Implementation | Justification |
|---|---|---|
| Minimal feature dimensionality | 33 features total; no raw TF-IDF vectors or learned embeddings as inputs. | Keeps the feature-to-sample ratio at 0.35, well below the critical threshold of 1.0. |
| No pretrained neural representations | Skip-gram word embeddings were evaluated and discarded due to noise on 190 texts. | Training from scratch on 190 documents produces noisy vectors; the embedding matrix alone exceeds training sample count in parameters. |
| Aggressive regularisation | LightGBM: depth-2, min_child_samples=15, reg_alpha=5, reg_lambda=10. SVM: C=0.05. LR: C=0.05. | Forces all models into the high-bias, low-variance regime appropriate for small datasets. |
| Delta and mean parameterisation | 24 raw per-text features become 12 deltas + 12 means. | Halves effective input dimensionality relative to raw concatenation without losing information. |
| Swap consistency at inference | Final prediction is the average of original and swapped predictions. | Eliminates positional bias and reduces prediction variance without additional learned parameters. |
| Conservative pseudo-labelling | Test pairs incorporated only at >= 98% confidence; ensemble retrained from scratch. | Minimises expected label error rate while exploiting high-confidence unlabelled data. |
| Simple ensemble averaging | Arithmetic mean of three model probabilities; no meta-learner. | A stacking meta-learner requires fitting on OOF predictions; arithmetic mean has zero additional parameters. |
| Repeated cross-validation | 5 folds x 3 repeats = 15 train/validation configurations. | Provides stable, low-variance OOF accuracy estimates on a dataset too small for single-fold CV. |
| Complementary model selection | LightGBM (tree), SVM (kernel), LR (linear). | Three distinct inductive biases ensure partially independent errors — the prerequisite for ensemble improvement. |
| Pooled corpus preprocessing | TF-IDF and LM trained on all 2326 texts, not just 190 training texts. | Prevents vocabulary gaps and IDF distortion at test time; uses no label information. |

## Results and Discussion

### Cross-Validation Performance

The OOF accuracy of the full pipeline — Phase 3, after Phase 5 pseudo-label retraining —
was consistently in the range of 87-89% across random seeds. The decomposition by model
and pipeline stage is as follows:

| Component                              | OOF Accuracy (approximate) |
|----------------------------------------|----------------------------|
| LightGBM alone                         | ~86%                       |
| SVM alone                              | ~83%                       |
| Logistic Regression alone              | ~84%                       |
| Ensemble (3-model average)             | ~87-88%                    |
| Ensemble + swap consistency            | ~88-89%                    |
| Full pipeline (+ pseudo-labelling)     | ~89-90%                    |

The consistent incremental improvement at each stage validates each design choice
independently. Swap consistency contributes approximately 1% and pseudo-labelling
approximately 2% to the baseline ensemble accuracy.

### Leaderboard Performance

| Metric               | Score   |
|----------------------|---------|
| Public Leaderboard   | 0.836   |
| Private Leaderboard  | 0.852   |

The improvement from public to private leaderboard (0.836 -> 0.852) indicates that the
model did not overfit to the public test subset. This is consistent with the strong
regularisation applied throughout the pipeline and confirms genuine generalisation.

### Gap Between CV Accuracy and Leaderboard Score

The CV OOF accuracy of approximately 87-89% is higher than the leaderboard score of
approximately 85%. This gap has two primary causes:

Distribution shift. The training and test corpora are drawn from the same general task
but may differ in topic distribution (e.g., the test set may contain articles on topics
underrepresented in training), in the LLMs used to generate hallucinated summaries, or
in article length distribution. All of these factors affect feature values at test time
relative to training-time expectations.

CV optimism. Even with repeated stratified CV, OOF accuracy slightly overestimates true
generalisation performance because the feature extraction and preprocessing steps (TF-IDF
fitting, LM training) see the full corpus including training labels, introducing a mild
form of leakage. In a perfectly rigorous implementation, these steps would be repeated
within each CV fold. The observed gap of 2-3 percentage points is within the expected
range for a small dataset with mild distribution shift and does not indicate systematic
overfitting.

### Error Analysis

Post-competition analysis using the _analyze.py script compared predictions across model
variants and identified the following systematic error patterns:

Short texts (fewer than 10 words). Perplexity estimates are unreliable for very short
sequences because the language model cannot aggregate meaningful statistics from fewer than
three trigram transitions. These samples are disproportionately represented in the error set.

Texts with unusual formatting. Articles with extensive mathematical notation, specialised
Unicode characters, or non-standard punctuation caused tokenisation artefacts that distorted
sentence-level features (n_sents, avg_sent_len, longest_sent).

Mid-length texts (100-300 words). These benefited most from pseudo-labelling. The baseline
model had moderate confidence (0.6-0.8) on many such pairs, and the retrained model — with
additional pseudo-labelled examples in this length range — became substantially more accurate.

## Potential Extensions

The following directions were identified as promising but not pursued within the competition
constraints. They are ordered by estimated impact on leaderboard score.

### Compression-Based Features (Normalised Compression Distance)

Normalised Compression Distance (NCD) is a parameter-free, universal similarity measure
based on Kolmogorov complexity, approximated via standard file compression algorithms:

    NCD(A, B) = (C(AB) - min(C(A), C(B))) / max(C(A), C(B))

where C(X) is the compressed size of string X using gzip or lzma. In separate experiments,
a pure NCD + k-nearest-neighbour approach achieved 0.843 on the public leaderboard —
competitive with the full feature-engineered pipeline. Integrating NCD as a fifth tower
(contributing one or two features) is estimated to push the combined score above 0.86.

### Character-Level Language Model

The current word-level Kneser-Ney LM is sensitive to tokenisation decisions and
out-of-vocabulary terms. A character-level KN model would eliminate vocabulary gaps
entirely and may capture surface-level hallucination artefacts — unusual character
sequences, formatting errors, repeated subword patterns — that the word-level model misses.

### Multi-Round Pseudo-Labelling with Decreasing Threshold

A confidence schedule of 0.98 -> 0.95 -> 0.90 across three rounds could add substantially
more pseudo-labelled samples to the training set. The primary risk — error amplification
across rounds — can be partially mitigated by monitoring the overlap between consecutive
rounds' label assignments and halting when inter-round inconsistency exceeds a threshold.

### Rule-Based Fallback for Very Short Texts

For texts with fewer than 10 tokens, perplexity is unreliable and most sentence-level
linguistic features are poorly estimated. A domain-specific rule (e.g., if one text is
less than 50% the length of the other, it is more likely truncated and hence hallucinated)
could improve accuracy on this specific subset at negligible additional cost.

### Bayesian Hyperparameter Optimisation

The current hyperparameters were set via manual grid search and domain knowledge. A
systematic Gaussian process optimisation over the joint hyperparameter space of all three
models could yield additional improvements, though the expected gain is modest (1-2%)
and the risk of finding hyperparameters that overfit the 95-sample OOF estimate is real.

## Conclusion

This pipeline demonstrates that principled, classical machine learning remains competitive
with neural approaches when data is extremely scarce and task-relevant features can be
derived from domain knowledge. The private leaderboard score of 0.852 was achieved through
five reinforcing design decisions:

Feature engineering over representation learning. A 33-dimensional hand-crafted feature
vector, grounded in the known linguistic properties of hallucinated text, outperformed
alternatives based on raw bag-of-words or trained word embeddings on this dataset size.
Every feature was selected for a clearly articulable discriminative rationale, not for
correlation in the training set.

Systematic regularisation throughout. Every learnable component — the three base classifiers,
their ensemble combination, and the pseudo-label selection criterion — was regularised to
prevent memorisation of the 95-sample training set. Regularisation parameters were chosen
to place all models firmly in the high-bias regime.

Unsupervised preprocessing on the full corpus. Using all available texts for TF-IDF and
language model training, without any label information, provided better-calibrated
preprocessing statistics and eliminated vocabulary gaps that would otherwise degrade
test-time performance.

Inference-time symmetry enforcement. Swap consistency eliminated positional bias and
reduced prediction variance at zero parameter cost, yielding a consistent 1-2% accuracy
gain across all experimental configurations.

Conservative semi-supervised learning. One round of pseudo-labelling at a 98% confidence
threshold safely expanded the effective training set by 50-100 samples, yielding a further
2% improvement that generalised cleanly to the private leaderboard.

Together, these principles constitute a reusable methodology for binary classification
tasks in low-resource settings where pretrained neural models are unavailable, prohibited,
or insufficiently constrained by the available training data. The key insight is that
deliberate, theoretically grounded feature engineering — combined with aggressive
regularisation and careful use of unlabelled data — can match or approach the performance
of much more complex systems, provided the features are well-matched to the specific
failure modes of the texts being classified.
